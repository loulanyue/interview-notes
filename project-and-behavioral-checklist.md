# 项目面与行为面准备清单

技术知识准备得再多，如果项目经历讲不清、行为题答得散，面试表现还是会打折。这份清单用于面试前快速整理“项目表达”和“行为问题”。

## 项目面准备

- 准备 2 到 3 个最能体现能力的项目
- 每个项目都能说清：
  - 背景和目标
  - 你的职责边界
  - 关键技术方案
  - 遇到的问题与权衡
  - 结果与量化收益

## 建议结构

可用一套固定顺序表达：

1. 这个项目要解决什么问题
2. 当时的约束是什么
3. 你负责哪一段
4. 做了哪些关键设计或优化
5. 最终效果如何
6. 如果重来一次会怎么改

## 高频行为题

- 你做过最有挑战的项目是什么？
- 你推动过哪些优化？
- 和同事意见不一致时怎么处理？
- 遇到线上故障时你是怎么排查的？
- 做过什么让系统更稳定或更高效的事情？

## 面试前自查

- 是否准备了可以量化的结果
- 是否能明确区分“团队成果”和“个人贡献”
- 是否能讲出一次失败或取舍，而不是只讲成功
- 是否能把技术细节说到“既具体又不跑偏”

## 使用建议

- 用口语化表达，不要背稿感太强
- 每个项目最好控制在 3 到 5 分钟版本
- 行为题优先用“情境 -> 行动 -> 结果 -> 反思”的方式回答
