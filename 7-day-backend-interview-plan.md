# 7 天后端面试复习计划

适合在面试前一周做高密度回顾。目标不是把所有内容重新看一遍，而是优先扫过高频主线、补齐最容易被追问的短板，并留出时间做口述训练。

## Day 1：计算机基础主线

- 操作系统：进程线程、内存管理、调度、IO 模型
- 网络：TCP 三次握手 / 四次挥手、拥塞控制、HTTP 与 HTTPS
- 输出目标：能用自己的话讲清 TCP 与 HTTP 的核心差异

## Day 2：Java 基础与集合

- 复习基础语法、面向对象、异常、泛型
- 重点看集合框架、HashMap、ConcurrentHashMap
- 输出目标：能解释常见集合在使用场景和线程安全上的取舍

## Day 3：并发与 JVM

- Java 并发、线程池、锁、CAS、AQS
- JVM 内存结构、GC、类加载、排查思路
- 输出目标：准备 2 到 3 个“线上问题如何排查”的回答模板

## Day 4：数据库

- SQL 基础、索引、事务、锁、隔离级别
- MySQL 调优思路、Redis 高频场景题
- 输出目标：能讲出慢 SQL、缓存穿透 / 击穿 / 雪崩的处理思路

## Day 5：框架与系统设计

- Spring / Spring Cloud / MyBatis 核心问题
- 系统设计基础、分布式、消息队列、安全
- 输出目标：准备一个完整的系统设计口述案例

## Day 6：算法与工具链

- LeetCode 高频题型、排序、链表、树、动态规划
- Linux / Git / Maven / Docker 常见命令和概念
- 输出目标：刷少量高频题，重点复盘思路表达而不是数量

## Day 7：模拟面试与复盘

- 用 [review-tips.md](./review-tips.md) 做一轮口述训练
- 按专题 README 快速扫一遍导航，看是否还有明显盲区
- 整理 3 类内容：熟练项、薄弱项、需要临时突击项

## 使用建议

- 每天先看专题 README，再进入具体笔记
- 每个专题至少做一次“不看原文复述”
- 如果时间紧，优先保证 Day 1 到 Day 5
